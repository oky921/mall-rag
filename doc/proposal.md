# 商城一期与二期提案

## 目标

在现有商品知识问答系统中增加一个可独立浏览的商城界面，并允许用户在智能助手与商城之间切换。后端使用本机 MySQL 保存商品。

## 本期范围

- 商品列表、分类筛选与关键词搜索。
- 首页推荐商品展示。
- 前端本地购物车数量与金额预览。
- 智能助手页和商城页双向切换。
- 应用首次启动时自动创建表并写入演示商品。

## 非目标

- 登录、收货地址、支付、订单履约和后台管理。
- 库存并发扣减和真实交易。
- 将 RAG 检索数据与商城主数据合并。

## 约束与假设

- Java 21、Spring Boot 4.1、React 和 Vite 保持不变。
- MySQL 默认连接 `root/123456@localhost:3306`，允许通过环境变量覆盖。
- 商品图片使用远程摄影资源，加载失败时显示本地样式占位。

## 验收标准

- `GET /api/store/products` 能从 MySQL 返回演示商品。
- `/mall` 能展示、搜索和筛选接口返回的商品。
- 商品可加入前端购物车预览，数量与总价会更新。
- `/` 与 `/mall` 可以相互切换。
- Maven 测试和前端生产构建通过。

## 二期目标

在一期商品列表基础上完成可落库的最小交易闭环：商品详情与 SKU 选择、演示用户数据库购物车、结算确认、模拟订单、事务内扣减 SKU 库存、清理已下单购物车项和订单结果查看。

## 二期范围

- 商品详情返回启用的 SKU 及 JSON 规格。
- 固定演示用户 ID `1`，用户来源封装为可替换组件，不实现登录注册。
- 购物车的查询、新增、改量和删除全部持久化到 MySQL。
- 从选中的购物车项创建模拟订单，保存商品与 SKU 快照。
- 下单事务内锁定 SKU、复核库存、扣减库存并清理对应购物车项。
- `/mall` 下提供详情、结算确认和订单结果视图。

## 二期非目标

- 真实支付、物流、优惠券、退款和订单后台。
- 多地址管理与登录鉴权。
- 跨服务分布式事务。

## 二期验收标准

- 商品详情能选择不同规格组合并显示对应价格和库存。
- 刷新页面后购物车仍存在，数量不能超过 SKU 库存。
- 成功下单后库存正确减少，对应购物车项被删除，并能查看订单快照。
- 库存不足或非法购物车项不会生成半成品订单。
- 后端测试和前端生产构建通过。

## 三期目标：订单中心

在模拟下单基础上补齐订单生命周期：用户能查看订单列表、筛选状态、模拟支付和取消待支付订单。取消订单必须在事务内只回补一次 SKU 库存。

## 三期范围与验收

- `/mall/orders` 展示固定演示用户的全部订单，并支持待支付、已支付、已取消筛选。
- 待支付订单可模拟支付，保存支付流水号和支付时间。
- 待支付订单可取消，保存取消时间并恢复 SKU 库存与销量。
- 重复支付或重复取消具备幂等结果；已取消订单不能支付，已支付订单不能取消。
- 并发支付/取消通过订单悲观锁串行化，库存只回补一次。
- 订单详情与下单结果页能够继续执行支付、取消并返回订单中心。
